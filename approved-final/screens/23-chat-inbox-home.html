Chat home should feel as natural as opening WhatsApp, but smarter for commerce.

Screen 23 keeps personal chat simple while attaching business, order, bill, proof and support context wherever needed. The user sees recent chats first, urgent order chats second, and can filter or start a chat in one tap.

Screen Contract Approved
EntryFrom Universal Chat action, unread notification, order badge, shop reply, delivery alert or support case.
User SeesWhatsApp-style recent inbox, unread badges, verified markers, call/video/media/file capability strip, priority order/support alerts and search.
DecisionOpen chat, filter All/Unread/People/Business/Orders/Support, start chat, call/video/media/file action, search, voice search or return to Mool.
BackendThread index service merges personal, business, order and support channels with real-time unread sync.
Tech StackWebSocket/SSE live updates, FCM push, local cache, optimistic read state, call signaling, media/file upload service and thread routing API.
TrustOrder chats carry proof packets, bills, refund status and support case IDs without exposing sensitive action outside the thread.
NextOrder/support thread routes to Screen 22; future people, business and support detail screens reuse the same thread contract.
FailureIf sync fails, show cached inbox with last updated time; if no chats exist, show start-chat empty state.
DockMool, People, Business, Orders, Support and Chat remain reachable without hiding the inbox list.
