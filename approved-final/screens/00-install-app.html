Screen 00

Install App

The user starts from the Play Store listing. This screen must create trust, explain the app simply, avoid false promises, and hand the user cleanly into the first app-open flow.

Approved

Purpose

Download trust before app trust.

The Play Store screen is not a marketplace screen. It should position MoolSocial as a direct operating socio-commerce app for social identity, daily needs, business growth, verified earning, money tools and chat.

This screen should not mention role switching. Work and business value can be visible, but every user is still entering as an individual consumer first.

Words To Avoid

  • Marketplace
  • Yellow pages
  • Demand Vault

About this app

MoolSocial helps consumers, families, businesses, creators, workers, captains, retailers and manufacturers connect social activity with daily needs, services, orders, payments, money tools, chat and verified earning opportunities where available.

Consumer first Business tools Proof-based earning Social commerce Money and chat

Screen Contract

Entry, install, and open-app handoff.

Entry condition User reaches Play Store from search, ad, QR, referral, website, or direct link.
Primary action Install. After install completes, the system button changes to Open.
Secondary action Read screenshots, app description, privacy details, ratings, and reviews.
Exit to next screen Open app -> Screen 01 First Open / Splash. Screen 01 is not designed yet.

Decision Rules

What this screen must communicate.

Positioning Direct operating socio-commerce app, not ecommerce listing, local directory, or marketplace.
User identity Every user starts as an individual consumer first; business, creator, captain and work layers activate only when verified.
Earn promise Say verified funded earning opportunities, never fixed income, guaranteed income, or free money.
Service promise Only show social, business, commerce, money, service, ride or medicine claims that are enabled in launch geography.
Trust promise Make payment, receipt, proof, refund, and support expectations visible in screenshots or copy.

Data Needed

Production inputs before launch.

  • Final MoolSocial app name, wordmark app icon, package name, and Play Store short description.
  • Approved screenshots for social, daily needs, business, earn, money, services, and chat.
  • Privacy policy URL, support email, and developer identity.
  • Age rating, data safety declarations, and permissions list.
  • Launch city and feature availability claims that match real operational readiness.

Compliance Gates

Must pass before production.

✓ No unsupported bill, ride, medicine, or local service claim appears in public listing.
✓ No outside-link or webview dependency is promised as a completed MoolSocial service.
✓ Income wording is outcome-funded and proof-based, not fixed salary or free money.
✓ Privacy policy and data safety disclosures match actual app behavior.

Regression Lock

Do not move to Screen 01 until this is true.

✓ Final Play Store title and short description approved.
✓ Install -> Open -> first app screen handoff defined.
✓ Feature claims match MVP readiness and geo activation rules.
✓ Production test plan includes install/open smoke test and store-listing content review.
