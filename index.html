Build Rule

Foundation first, production code second.

The HTML files in this folder are not the final app. They are the locked brand and UI/UX architecture reference for Codex before building the production frontend, backend APIs, and tests.

No screen should move into production until brand usage, entry conditions, visible UI, tap behavior, data needs, failure states, compliance gates, and regression checklist are written here.

Regression Discipline

Lock one screen before touching the next.

  • Brand foundation must remain stable across every screen.
  • Screen spec approved in this screenbook.
  • Production screen implemented against the approved spec.
  • API/data contract written before backend integration.
  • Happy, empty, loading, and error states tested.
  • Screenshot or visual baseline saved for regression checks.
  • Future changes must keep all locked tests passing.
