Screen Contract Entry, behavior and exit

Entry Condition

User taps Open after installing, taps the app icon, or opens a valid app link that reaches the app shell.

Primary Behavior

Auto-run boot checks and route forward. No user action is required unless a recoverable error appears.

Exit To Next Screen

Fresh install exits to Screen 02: First Setup. The next screen will handle language, location explanation and permission timing.

Routing States Production decision paths

Fresh Install

No user session exists. Route to Screen 02 First Setup after boot checks pass.

Returning Signed Out

Known device without valid session. Route to mobile login after first setup checks are complete.

Returning Signed In

Valid session. Route to Universal Super-App Screen or pending priority action.

Deep Link

Preserve target intent, complete auth/setup if needed, then continue to the target only if safe.

What This Screen Must Not Show Regression-sensitive

No Role Switch

No Personal, Retailer, Taxi, Creator, Business or Work mode selector. Roles activate later through verified context.

No Permission Prompt

No location, camera, microphone, contacts, messages or notification permission prompt appears on splash.

No Claims

No earning claim, marketplace claim, discount claim or unavailable service claim appears during splash.

Data Needed Inputs for production implementation

Device And App Inputs

  • App version, build channel and minimum supported version.
  • Device OS, locale, timezone and network state.
  • Install source, campaign/referral/QR code and deep link payload.
  • Existing auth token, refresh state and anonymous device identity.
  • Permission state only as device metadata, not as a permission request.

Remote Config Inputs

  • Maintenance state and feature gate status.
  • Launch geography and category readiness flags.
  • Critical compliance blocks.
  • Minimum app version and force-update URL.
  • First setup route decision.
Failure States Only if boot cannot continue

No Internet

Show "Check connection" and Retry. Do not erase install attribution or deep-link intent.

Update Required

Show Update App. Route to store. Do not allow unsupported build to continue.

Maintenance

Show short maintenance state with Retry and Help. Keep wording calm and non-technical.

Invalid Link

Continue as normal first open, but do not execute expired or unsafe deep-link action.

Compliance And Regression Lock Before Screen 02

Compliance Gates

  • Only approved MoolSocial logo and colours are used.
  • No sensitive permission prompt appears before explanation.
  • No user role is asked or implied.
  • No guaranteed earning, marketplace or unavailable service wording appears.
  • Only necessary technical boot checks run before consent-sensitive flows.

Regression Checklist

  • Open from Play Store routes to splash.
  • Fresh install routes to Screen 02.
  • Returning signed-in user can bypass setup only with valid session.
  • Deep-link intent is preserved through auth/setup.
  • Failure states show Retry, Update or Help without trapping the user.
