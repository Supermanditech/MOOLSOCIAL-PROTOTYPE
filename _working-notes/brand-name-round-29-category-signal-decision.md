# Brand Name Round 29 - Category Signal Decision

Working note only. Not final legal clearance.

## Latest Rejection

The user rejected "Seedha":

> People will question what is Seedha. Not workable.

This is an important correction.

The name cannot be:
- only invented
- only emotional
- only premium sounding
- only a quality word like direct/trusted/fast

The name must give a category signal.

## New Naming Rule

The name must answer this question quickly:

> What kind of app is this?

For SuperMandi, the public answer needs to feel like:

> This is one app for daily buying, selling, work, payment, service, social connection and local economic action.

That is a lot for one name to carry. So there are only three serious routes:

1. Keep **SuperMandi** and sharpen the public descriptor.
2. Create a compound name with a category signal.
3. Use a new short brand name, but pair it everywhere with a strong descriptor.

Without one of these, people will keep asking "what is it?"

## Route 1 - Keep SuperMandi As App Name

### SuperMandi

Public perception:
- super = bigger/all-in-one
- mandi = Indian economic place
- together = all-in-one India commerce/economy app

Usage:
- "SuperMandi pe dekh le."
- "SuperMandi se payment bhej."
- "SuperMandi pe kaam mil gaya."
- "SuperMandi pe order daal."
- "SuperMandi pe post kar."

Why this may actually be strongest:
- It already has category signal.
- It already connects to the company name.
- It sounds Indian enough and broad enough.
- It can include commerce, work, local services and value-chain logic.
- It avoids the "what is Seedha?" problem.

Main risk:
- People may assume marketplace/mandi/listing if the UI is weak.

How to solve:
- Do not change name.
- Change first impression:
  - "India's direct social-commerce operating app"
  - Search / Ask / Scan / Voice visible first
  - Chat, Pay, Work, Social and Commerce visible together
  - Decision cards instead of product grids

This means the brand architecture does the explanation, not only the name.

## Route 3 - Short Brand + Strong Descriptor

If we eventually choose a short brand name, it must not appear alone in Play Store or onboarding.

Example structure:

> [Brand] - Social, Shop, Work & Pay

or

> [Brand] - India's Daily Commerce App

or

> [Brand] - Buy, Work, Pay, Connect

This route makes names like Tavo/Yonda-type names possible, but only if the descriptor does the explanation.

However, the user currently wants the name itself to carry meaning, so Route 3 is not preferred now.

## Honest Recommendation

At this point, if the name must be understandable from first impression, **SuperMandi** is still stronger than all invented alternatives.

Why:
- The public can understand "super" and "mandi".
- It gives a broad economic category signal.
- It connects with the registered company name.
- It can become a premium mass India brand if the UI makes it clear this is not a marketplace.

The real issue may not be the name.

The real issue is the first public explanation:

> SuperMandi - Buy, Work, Pay, Connect

or

> SuperMandi - India's Direct Social-Commerce App

or

> SuperMandi - One App For Daily Economy
