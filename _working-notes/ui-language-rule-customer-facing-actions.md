# UI Language Rule: Customer-Facing Actions Only

Date: 2026-07-09

Locked working rule for future screens:

- Do not use internal planning words inside app-facing UI.
- Avoid labels such as master, module, system, layer, route-specific, workflow, decision engine, MVP, backend, contract, or planning terms inside the phone screen.
- Use direct customer action words instead: Deliver, Scan QR, Find, Offers, Order Food, Book Table, Tiffin, Pay, Chat, Track, Reorder, Cancel, Refund.
- Keep the first viewport clean and actionable. If a screen needs many choices, show a compact primary choice first and reveal detail after tap.
- Each app-facing card should answer what the user can do now, price/cost where relevant, time/slot, trust/proof, cancellation/refund, and the next action.
- Planning terms can remain only in screenbook notes, contracts, or working documents, not in production-facing phone UI.
