## 2. Corrected Product Boundary

`Earn` and `My Work` are different products.

```text
Earn = funded opportunity discovery and application
My Work = user-type onboarding, workspace creation and workspace operations
```

Earn must never become the business/provider registration system.

My Work must never become only a freelancer screen.

My Work is the universal operating gateway through which every product seller, supplier, manufacturer, service provider, professional, creator, worker, mobility partner, organization or institution creates and operates a verified workspace.

## 3. Non-Negotiable Identity Rule

```text
One person
One personal MoolSocial account
Consumer and social access always active
Zero or more separately verified workspaces
One active workspace at a time inside My Work
```

Every user remains a consumer and social user first.

Creating a workspace adds an operating layer. It never removes or replaces personal access to Social, Buy, Eat, Ride, Book, Pay, Work or Chat.

A single person may simultaneously own or operate multiple workspaces, for example:

```text
Personal account
Retail shop workspace
Creator workspace
Freelancer workspace
Taxi captain workspace
```

Each workspace keeps separate verification, permissions, catalog or services, availability, operations, compliance, reviews, money and settlement.

## 4. Screen 69 Is My Work

Screen 69 must be replaced with the universal `My Work` home and workspace gateway.

It is not Freelancer Work setup.

It is not Creator Work setup.

It is not a giant role directory.

### New user state

A user with no workspace sees:

```text
My Work
Start My Work
Create the work or business profile you operate
```

The next interaction progressively asks what the person wants to operate. It must not expose every subtype at once.

### Existing user state

A user with one workspace sees that workspace directly with its current urgent actions.

A user with multiple workspaces sees:

```text
Current workspace
Other workspaces
Add Another Work
```

Workspace switching happens deliberately inside My Work. There is no visible role-switch strip on the universal consumer screen.

### Contextual entry from an opportunity

If a consumer with no workspace reaches My Work after tapping Apply, Screen 69 keeps the opportunity saved and explains that a work profile is needed. It may recommend the appropriate entry door, but it must not silently create a role.

If a user already has a verified workspace, Earn and opportunity application use that verified operating identity according to eligibility rules. The user is not forced to create a duplicate workspace.

The exact Apply routing must be finalized after the Screen 69 My Work states are approved.

## 10. Workspace Activation State Machine

```text
DRAFT
-> OWNER / OPERATOR IDENTITY
-> TYPE AND SUBTYPE
-> ROLE-SPECIFIC VERIFICATION
-> SELL / SERVICE SETUP
-> BUY / PROCUREMENT SETUP
-> PRICE / AVAILABILITY
-> FULFILMENT / DELIVERY / SLOT READINESS
-> PAYMENT / INVOICE / SETTLEMENT
-> COMPLIANCE AND SUPPORT GATES
-> ACTIVE, LIMITED or PENDING
```

Registration alone never publishes a product, menu, service, appointment, ride, delivery capacity or offer.

Example restaurant go-live:

```text
Restaurant workspace verified
-> outlet and food compliance verified
-> menu, price and availability configured
-> order acceptance configured
-> table/takeaway/delivery modes configured
-> payment, cancellation and support rules configured
-> geo and fulfilment ready
-> menu becomes eligible for Eat decision cards
```

Example retailer go-live:

```text
Retailer workspace verified
-> shop and owner verified
-> retail/wholesale modes configured
-> products, pack, price and stock configured
-> counter pickup/home delivery configured
-> refund and stock-confirmation rules configured
-> catalog becomes eligible for Buy decision cards
```

## 12. My Work Daily Experience

My Work is not only registration. After activation, it becomes the role-specific operating home.

Every workspace home should answer:

```text
What needs attention now?
What can I sell or serve now?
What must I procure now?
Who is waiting for me?
What delivery, booking, order, task or payment is urgent?

```

The first workspace screen should show urgent actions and current state, not static features or a marketing dashboard.

All verified workspace types retain access to Earn, because MoolSocial or another verified workspace may publish funded work relevant to them.

## 17. Locked Summary

MoolSocial is one consumer-first super-app with many verified operating workspaces behind My Work. Every provider workspace owns both its sell/service side and its buy/procurement side. The products, menus, services, slots, rides, supplies and professional services configured inside those workspaces become eligible for consumer-facing outcome cards only after verification, readiness, compliance, fulfilment, payment and support gates pass. My Work is therefore the bridge between the consumer super-app and the full supply, service and value-chain operating system.
